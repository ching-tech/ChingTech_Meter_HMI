Master / Slave 雙機部署

0架構概覽

💡
程式碼兩台完全相同，唯一差異在 config.json 的設定。

1安裝 Python 環境（兩台都要）

下載 Python 3.10 以上版本（建議 3.13），安裝時勾選 "Add to PATH"。

CMD / PowerShell# 安裝必要套件
pip install nicegui pymcprotocol
💡
藍芽通訊使用 Windows 原生 Bluetooth Socket（socket.AF_BLUETOOTH + BTPROTO_RFCOMM），Python 3.9+ 內建支援，不需要安裝額外藍芽套件。

2複製程式檔案

將以下檔案複製到兩台電腦的同一個資料夾（例如 D:\ChingTech_Meter_HMI）：

ChingTech_Meter_HMI/ ├── main.py ├── config.py ├── bluetooth_comm.py ├── plc_comm.py ├── network_comm.py ├── measurement.py ├── meter32x32.ico ├── config.json ← 每台電腦要各自設定！ ├── logs/ ← 自動產生，不需複製 │ ├── slave_test/ 不需複製 ├── openspec/ 不需複製 ├── reference/ 不需複製 ├── __pycache__/ 不需複製 └── .git/ 不需複製

3設定 config.json（關鍵步驟）

電腦 A — Master
{
  "simulation_mode": false,

  "plc": {
    "enabled": true,
    "ip_address": "192.168.1.10",
    "port": 5000
  },

  "network": {
    "mode": "master",
    "master_ip": "192.168.1.100",
    "port": 5001,
    "slave_meter_offset": 6
  },

  "bluetooth": {
    "enabled": true,
    "device_addresses": [
      "AA:BB:CC:DD:EE:01",
      "AA:BB:CC:DD:EE:02",
      ... Master 負責的 6 支耳溫槍
    ]
  },

  "measurement": {
    "channel_enabled": [
      false, false, false, false, false, false,
      true,  true,  true,  true,  true,  true
    ]
    // Master 管 CH 7-12（後 6 個）
  }
}
電腦 B — Slave
{
  "simulation_mode": false,

  "plc": {
    "enabled": false,
    // Slave 不直接連 PLC
  },

  "network": {
    "mode": "slave",
    "master_ip": "192.168.1.100",
    // ↑ 填 Master 的實際 IP！
    "port": 5001,
    "slave_meter_offset": 6
  },

  "bluetooth": {
    "enabled": true,
    "device_addresses": [
      "AA:BB:CC:DD:EE:07",
      "AA:BB:CC:DD:EE:08",
      ... Slave 負責的 6 支耳溫槍
    ]
  },

  "measurement": {
    "channel_enabled": [
      true,  true,  true,  true,  true,  true,
      false, false, false, false, false, false
    ]
    // Slave 管 CH 1-6（前 6 個）
  }
}

4網路設定

  1. 兩台電腦必須在同一個區域網路（同一個 switch / router）
  2. 建議設定靜態 IP：
    • Master: 192.168.1.100
    • Slave: 192.168.1.101
  3. Master 電腦的 Windows 防火牆必須開放 TCP port 5001
⚠️
防火牆未開放 port 5001 是最常見的連線失敗原因！
PowerShell（管理員）— Master 電腦執行# 開放防火牆 port 5001
netsh advfirewall firewall add rule name="ChingTech HMI" dir=in action=allow protocol=TCP localport=5001

5建立啟動捷徑

在兩台電腦的程式資料夾中各建一個 啟動系統.bat：

啟動系統.bat（兩台相同）@echo off
cd /d "%~dp0"
start "" python main.py
exit

程式會自動讀取同目錄下的 config.json，依據 mode 決定 Master 或 Slave 行為。

6啟動順序

1
先啟動 Master
開始監聽 port 5001
➡
2
再啟動 Slave
自動連線至 Master

Slave 斷線後每 3 秒自動重連，無需手動介入。
