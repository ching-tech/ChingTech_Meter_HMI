## 1. 系統總覽

「擎添耳溫槍探頭套檢測系統」是一套工業級自動化 HMI 軟體，用於檢測耳溫槍探頭套品質。系統透過**藍芽**連接最多 12 支耳溫槍、與 **PLC** 整合產線觸發信號，並以 **Master-Slave 雙機架構**分散管理通道。

### 核心功能

- **12 通道同時監測** — Master 負責 6 通道 (CH1/3/5/7/9/11)，Slave 負責另外 6 通道 (CH2/4/6/8/10/12)
- **自動量測判定** — 空槍值 → 量測值 → 誤差計算 → PASS/FAIL 自動判定
- **PLC 整合** — 接收產線觸發信號 (D500/D515)，回寫判定結果 (D501~D512) 與異常狀態 (D513)
- **異常即時警報** — 藍芽斷線、溫度異常、連續無耳溫套、PLC/網路異常，紅色閃爍橫幅通知
- **CSV 完整記錄** — 每次量測自動記錄 54 欄位，含溫度、耳套狀態、OK/NG 計數
- **異常歷史記錄** — 所有異常事件自動寫入 Alarm CSV，一天一個檔案

## 2. 系統架構

### 硬體架構

系統由以下硬體組成：

1. **PLC (三菱 FX5U)** — 透過 MC Protocol 3E (TCP) 與 Master 通訊，負責產線觸發與結果回收
2. **Master 電腦** — 運行主程式，管理 6 通道藍芽耳溫槍，負責判定邏輯與 CSV 記錄
3. **Slave 電腦** — 運行從機程式，管理另外 6 通道藍芽耳溫槍，將資料轉發給 Master
4. **耳溫槍 x12** — 透過 Bluetooth RFCOMM SPP 協定連線

連線方式：
- PLC ↔ Master：TCP (MC Protocol 3E)，預設 IP 192.168.1.10 Port 5000
- Master ↔ Slave：TCP Port 5001
- Master ↔ 耳溫槍 x6：Bluetooth RFCOMM
- Slave ↔ 耳溫槍 x6：Bluetooth RFCOMM

### 軟體模組

| 模組 | 檔案 | 功能 |
|------|------|------|
| 主程式 / UI | `main.py` | NiceGUI 介面、事件處理、異常偵測 |
| 藍芽通訊 | `bluetooth_comm.py` | RFCOMM SPP 連線、封包解析、自動重連 |
| PLC 通訊 | `plc_comm.py` | 三菱 MC Protocol 3E、讀寫 D 暫存器 |
| 網路通訊 | `network_comm.py` | Master-Slave TCP 同步、指令與資料傳輸 |
| 量測邏輯 | `measurement.py` | PASS/FAIL 判定、CSV 批次記錄 |
| 設定管理 | `config.py` | JSON 設定檔讀寫、參數定義 |

### Master vs Slave 差異

| 功能 | MASTER | SLAVE |
|------|--------|-------|
| 管理通道 | CH1, CH3, CH5, CH7, CH9, CH11（內部編號 1~6） | CH2, CH4, CH6, CH8, CH10, CH12（內部編號 7~12） |
| PLC 通訊 | 有 | 無 |
| 量測判定 | 完整 PASS/FAIL 判定 | 僅收集資料轉發給 Master |
| CSV 記錄 | 有（量測批次檔） | 無 |
| 異常偵測 | 全面偵測（溫度異常、連續無套、空槍超限） | 僅藍芽連線狀態 |
| 手動觸發按鈕 | 有（空槍量測、溫度量測） | 無 |
| OK/NG 計數顯示 | 有 | 無 |
| PLC 監控面板 | 有 | 無 |
| Web UI Port | 8080 | 8081 |
| TCP 角色 | Server（監聽 Port 5001） | Client（連線到 Master IP） |

## 3. 啟動與關閉

### 啟動前準備

啟動前請確認以下事項：

- 電腦的**藍芽**已開啟，且 12 支耳溫槍已開機
- **PLC** 已上電且網路線已連接（Master 端）
- Master 與 Slave 兩台電腦在**同一網段**，可互相 ping 通
- `config.json` 中的 IP 位址、MAC 位址正確

### 啟動步驟

1. **先啟動 Slave**（建議）：執行 `python main.py` 或雙擊捷徑。Slave 會在 Port 8081 開啟 UI 並等待 Master 連入。

2. **再啟動 Master**：執行 `python main.py` 或雙擊捷徑。Master 會在 Port 8080 開啟 UI，並主動連線到 Slave 與 PLC。

3. **確認連線狀態**：等待畫面頂部的 PLC 與網路指示燈變成綠色。各通道的 BT 圖示也應逐步變成藍色（已連線）。

4. **系統就緒**：畫面右上方顯示「運行中」（綠色）即表示系統已準備好接收 PLC 觸發。

### 關閉程式

直接關閉視窗即可。系統會自動執行清理動作：

- 關閉所有藍芽 Socket 連線
- 停止 PLC 監控
- 斷開 Master-Slave 網路

**注意**：關閉 Master 前，建議先確認目前無正在進行的量測流程（量測狀態為「待機中」）。若在量測過程中強制關閉，該筆資料可能不會寫入 CSV。

## 5. 量測流程

每次完整量測包含兩個階段：**空槍量測**（記錄基準值）→ **溫度量測**（記錄實測值並判定）。

### 完整流程

#### 第一階段：空槍量測

1. PLC 發送 D515=1（空槍觸發），或操作員手動按「空槍量測」按鈕
2. Master 通知 Slave 執行空槍量測（發送 `request_empty` 指令）
3. 所有啟用通道同時發送 CD 指令（要求耳溫槍量測）
4. 等待 `empty_collect_delay` 秒（預設 0.5 秒）讓資料回傳
5. 收集所有通道的空槍值
6. 檢查空槍值是否在合理範圍內（空槍上限/下限）
7. 將空槍資料寫入 CSV（一列 "empty" 記錄）
8. 清除 PLC D515（D515=0 表示空槍完成）
9. 量測狀態變為「空槍完成」

#### 第二階段：溫度量測

1. PLC 發送 D500=1（量測觸發），或操作員手動按「溫度量測」按鈕
2. Master 通知 Slave 執行溫度量測（發送 `request_measure` 指令）
3. 所有啟用通道同時發送 CD 指令
4. 等待 `measure_collect_delay` 秒讓資料回傳
5. 收集所有通道的量測值（本機 + Slave 網路資料）
6. 執行異常偵測（溫度異常、連續無套）
7. 計算誤差 = 量測值 - 空槍值
8. 依判定公式判定 PASS 或 FAIL
9. 寫入判定結果到 PLC D501~D512（0=不使用, 1=FAIL, 2=PASS）
10. 更新 PLC OK/NG 計數（D517~D540）
11. 將量測資料寫入 CSV
12. 清除 PLC D500
13. 量測狀態變為「量測完成」

### 判定公式

- **誤差** = 量測值 - 空槍值
- **判定條件**：`-|誤差下限| ≤ 誤差 ≤ +|誤差上限|`
  - 符合 → **PASS**（PLC 寫入 2）
  - 不符合 → **FAIL**（PLC 寫入 1）
  - 通道停用 → 不判定（PLC 寫入 0）

**範例**：誤差上限 = 0.30，誤差下限 = 0.50 → 合格範圍 = -0.50 ~ +0.30°C

### 判定模式

| 模式 | 行為 | 用途 |
|------|------|------|
| 正常判定 | 依公式計算 PASS/FAIL | 正式生產使用 |
| 強制 OK | 不論誤差值，全部判定為 PASS | 系統測試 / 暖機用 |
| 強制 NG | 不論誤差值，全部判定為 FAIL | 系統測試用 |

判定模式可在進階設定中切換，切換後立即生效。

### 手動觸發

MASTER 畫面下方有「手動觸發」面板，提供兩個按鈕：

- **空槍量測** — 等同 PLC D515=1 觸發。如果 PLC 已連線，會先寫入 D515 再由 PLC 回送觸發；如果 PLC 未連線，直接觸發。
- **溫度量測** — 等同 PLC D500=1 觸發，同上邏輯。

手動觸發主要用於調試和無 PLC 環境下測試，正式生產以 PLC 觸發為主。

## 6. 批次與記錄

### 量測 CSV 批次檔

- **檔案位置**：`D:\logs\{時間戳}.csv`
- 每個批次一個檔案，檔名為建立時的時間戳記（例：`20260312143025.csv`）
- 程式重啟時會自動沿用上次的批次檔案繼續寫入

#### CSV 欄位結構（54 欄）

| 欄位範圍 | 內容 | 說明 |
|----------|------|------|
| A 欄 | 種類 | `empty` = 空槍量測 / `{D516值}` = 溫度量測（顯示測試週期數） |
| B ~ M 欄 | scan1 ~ scan12 | 12 通道溫度值（按 CH1~CH12 顯示名稱排序） |
| N 欄 | Time | 量測時間 `YYYY/MM/DD HH:MM:SS` |
| O ~ P 欄 | 誤差上/下限 | 本次使用的判定設定值 |
| Q ~ AB 欄 | scan1~12 cover | 耳套狀態（`1111` = 有耳溫套 / `0000` = 無耳溫套） |
| AC ~ AN 欄 | scan1~12 OK | 各通道 OK 累計次數 |
| AO ~ AZ 欄 | scan1~12 NG | 各通道 NG 累計次數 |
| BA ~ BB 欄 | TOTAL OK / NG | 全通道 OK/NG 加總 |

### 換批操作

1. 確認目前量測狀態為「待機中」
2. 點擊頂部的「換批」按鈕
3. 系統會自動建立新 CSV 檔案，並將 PLC 的 OK/NG 計數（D517~D540）全部歸零
4. 頂部批次檔名會更新為新檔名

### 異常記錄（Alarm CSV）

- **檔案位置**：`D:\logs\Alarm\alarm_{日期}.csv`（每天一個檔案）
- 每筆記錄包含三個欄位：

| 欄位 | 說明 |
|------|------|
| 日期時間 | YYYY-MM-DD HH:MM:SS |
| 異常類型 | 藍芽斷線 / 溫度異常 / 連續無套 / PLC異常 / 網路異常 / 通道不一致 / 空槍異常 |
| 詳細內容 | 具體的異常描述文字 |

如果檔案被 Excel 鎖定（如正在檢視中），系統會自動寫入備用檔案 `alarm_{日期}_1.csv`，不會遺失資料。

### Debug 記錄

系統所有 console 輸出同步寫入 `logs/debug_{日期}_{時間}.txt`，可用於後續問題排查。每次啟動程式會建立新的 debug 檔案。

## 7. 異常警報處理

系統會自動偵測多種異常，並在畫面上方顯示紅色閃爍橫幅。所有異常都會記錄到 Alarm CSV。

### 7.1 藍芽斷線

- **觸發條件**：任何啟用通道的藍芽狀態變為 DISCONNECTED 或 ERROR
- **影響**：PLC D513 對應位元設為 1，該通道無法收集量測數據
- **處理方式**：系統會自動重連（漸進式冷卻），正常情況下等待即可恢復。若持續斷線，請檢查耳溫槍電源與藍芽配對。
- **自動恢復**：重新連線後自動清除警報與 D513 位元

### 7.2 溫度異常

- **觸發條件**：量測溫度超出設定範圍（溫度異常上限 ~ 下限），需在進階設定中開啟「溫度異常」開關
- **影響**：PLC D513 bit12 設為 1，異常通道列變紅色高亮
- **處理方式**：檢查耳溫槍感應器是否正常，或環境溫度是否影響量測
- **自動恢復**：下次量測所有通道恢復正常範圍時自動解除

### 7.3 連續無套

- **觸發條件**：同一通道連續 N 次偵測到「無耳溫套」(0000)，N 為設定的觸發次數，需開啟「連續無套」開關
- **影響**：PLC D513 bit13 設為 1，無套計數欄變橘色
- **處理方式**：確認探頭套是否正確安裝
- **自動恢復**：偵測到有耳溫套 (1111) 時，該通道計數歸零

### 7.4 空槍值超限

- **觸發條件**：空槍量測值超出空槍上限/下限範圍
- **影響**：PLC D513 bit14 設為 1
- **暖槍特殊處理**：暖槍中（PLC D542=1）時容忍前 2 次超限，第 3 次才發出警報。暖槍結束（D542=0）後超限計數自動歸零。
- **處理方式**：確認耳溫槍是否正常，可能需要繼續暖機

### 7.5 PLC 連線異常

- **觸發條件**：PLC TCP 連線中斷
- **處理方式**：檢查 PLC 電源、網路線、IP 設定
- **自動恢復**：PLC 通訊模組會自動重連

### 7.6 網路異常

- **觸發條件**：Master-Slave TCP 連線斷開
- **處理方式**：檢查網路線、IP 設定、防火牆規則
- **自動恢復**：網路管理器會自動重連

### 異常復歸操作

點擊畫面右上方的「異常復歸」按鈕（黃色），或 PLC D541 升沿觸發，將執行以下動作：

1. 清除 PLC D513 所有位元（異常掩碼歸零 0x0000）
2. 重置溫度異常與連續無套狀態旗標
3. 清除所有通道的無套計數歸零
4. 移除通道紅色高亮
5. 隱藏警報橫幅

## 8. 進階設定面板

點擊右上角的齒輪圖示開啟右側設定面板。

### 密碼保護

部分設定需要管理密碼才能修改。輸入密碼後點擊「登入」即可解鎖受保護的設定區域。登出後設定區域會重新隱藏，防止未授權修改。

### 系統設定（需登入）

| 設定項 | 說明 | 預設值 |
|--------|------|--------|
| 誤差上限 | PASS/FAIL 判定的正向容許誤差 | +0.50 °C |
| 誤差下限 | PASS/FAIL 判定的負向容許誤差 | -0.50 °C |
| 空槍上限 | 空槍值的合理上限，超出會發警報 | 40.0 °C |

## 9. 藍芽連線管理

系統使用 Windows 原生藍芽 RFCOMM 協定，透過 SPP (Serial Port Profile) 與耳溫槍通訊。

### 連線流程

1. **啟動** — BluetoothManager 啟動
2. **暖機** — 藍芽堆疊暖機（短暫探測連線 3 秒，觸發 Windows 藍芽堆疊初始化）
3. **並行初始連線** — 所有啟用通道同時建立連線（每通道一條 Thread），最多等待 connect_timeout 秒
4. **進入監控迴圈** — 持續偵測是否有通道斷線
5. **斷線重連** — 偵測到斷線通道時，同時並行重連，每輪間隔 reconnect_interval 秒
6. **漸進式冷卻** — 連續失敗 3 次以上時啟用冷卻機制，避免頻繁無效重連

### 三個關鍵時間參數

| 參數 | 預設值 | 作用 |
|------|--------|------|
| connect_timeout | 10 秒 | 單次 socket.connect() 最長等待時間。超時即視為連線失敗。 |
| reconnect_interval | 5 秒 | 每輪重連嘗試後的固定等待間隔。無論成敗都會等待此時間。 |
| cooldown（冷卻） | min(失敗次數×5, 30) 秒 | 連續失敗 ≥ 3 次後的額外等待。第 6 次以上固定 30 秒封頂。 |

### 冷卻時間對照表

| 連續失敗次數 | 冷卻時間 | 單輪最長等待 (timeout+interval+cooldown) |
|-------------|---------|----------------------------------------|
| 第 1~2 次 | 0 秒（立即重試） | 10 + 5 + 0 = 15 秒 |
| 第 3 次 | 15 秒 | 10 + 5 + 15 = 30 秒 |
| 第 4 次 | 20 秒 | 10 + 5 + 20 = 35 秒 |
| 第 5 次 | 25 秒 | 10 + 5 + 25 = 40 秒 |
| 第 6 次以上 | 30 秒（上限） | 10 + 5 + 30 = 45 秒 |

### BT 圖示顏色

| 顏色 | 狀態 | 說明 |
|------|------|------|
| 藍色 | CONNECTED | 已連線，可正常收發資料 |
| 黃色 | CONNECTING | 正在嘗試連線 |
| 紅色 | ERROR | 連線失敗或異常斷開 |
| 灰色 | DISCONNECTED | 未連線 |

### 通訊協定

耳溫槍設備類型代碼為 `REEB0001`，封包格式：

- **封包結構**：`[STX=0x02][Data][CheckSum][ETX=0x03][EOT=0x04]`
- **CheckSum**：payload 所有 byte 的 XOR
- **DB 封包**（量測資料）：包含設備 ID、溫度值、估算溫度、耳套狀態（1111/0000）、溫度模式
- **CB 封包**（ACK 回應）：量測確認回應
- **CD 封包**（量測請求）：主動要求耳溫槍執行量測

## 10. PLC 通訊

MASTER 透過 MC Protocol 3E (TCP) 與三菱 FX5U PLC 通訊。讀取週期為 100ms，批次讀取 D500~D542 共 43 個 word。

### D 暫存器配置表

| 暫存器 | 名稱 | 方向 | 說明 |
|--------|------|------|------|
| D500 | 量測觸發 | PLC→HMI (讀) / HMI→PLC (寫0) | 升沿 (0→1) 觸發溫度量測，HMI 完成後寫 0 |
| D501~D512 | 判定結果 | HMI→PLC | 0=不使用, 1=FAIL, 2=PASS（CH1~CH12） |
| D513 | 異常掩碼 | HMI→PLC | 位元掩碼，各位元代表不同異常類型 |
| D514 | PC 心跳 | HMI→PLC | 0/1 每秒交替，PLC 可偵測 HMI 是否存活 |
| D515 | 空槍觸發 | PLC→HMI (讀) / HMI→PLC (寫0) | 升沿觸發空槍量測 |
| D516 | 測試週期 | PLC→HMI | 累計測試次數 |
| D517~D528 | OK 計數 | HMI→PLC | CH1~CH12 各通道合格次數 |
| D529~D540 | NG 計數 | HMI→PLC | CH1~CH12 各通道不合格次數 |
| D541 | 異常復歸 | PLC→HMI | 升沿觸發異常復歸（與 HMI 按鈕相同效果） |
| D542 | 暖槍訊號 | PLC→HMI | 0=正常模式, 1=暖槍中 |

### D513 異常掩碼詳解

| 位元 | 內容 |
|------|------|
| bit 0 ~ bit 11 | CH1~CH12 藍芽連線異常（各通道獨立） |
| bit 12 | 溫度異常（任一通道量測溫度超出設定範圍） |
| bit 13 | 連續無套異常（任一通道連續無耳溫套超過門檻） |
| bit 14 | 空槍值超限異常（空槍量測值超出上下限） |

PLC 監控面板以十六進位顯示 D513 的值，例如：
- `0x0000` = 無異常
- `0x0004` = bit2 = CH3 藍芽斷線
- `0x1000` = bit12 = 溫度異常
- `0x2000` = bit13 = 連續無套異常

### PLC 連線驗證

系統啟動後會持續以 100ms 間隔輪詢 PLC。同時透過 D514 心跳信號（每秒切換 0/1）讓 PLC 知道 HMI 程式仍在運作中。PLC 可利用此心跳偵測 HMI 離線。

## 11. Master-Slave 網路

系統使用 TCP 在 Master 與 Slave 之間同步資料與指令。

### 角色與連線

| 項目 | MASTER | SLAVE |
|------|--------|-------|
| TCP 角色 | Server（監聽 Port 5001） | Client（連線到 Master IP:5001） |
| 建議啟動順序 | 第二（等 Slave 準備好再啟動） | 第一 |

### 指令與資料流

**Master → Slave 的指令：**

| 指令 | 觸發時機 | Slave 動作 |
|------|---------|-----------|
| `request_empty` | D515=1 空槍觸發 | 執行空槍量測，回傳溫度資料 |
| `request_measure` | D500=1 量測觸發 | 執行溫度量測，回傳溫度資料 |
| `sync_bt_status` | 網路連線建立後 | 重送所有通道的藍芽狀態與通道啟用設定 |

**Slave → Master 的資料：**

Slave 透過 JSON 封包將以下資訊即時傳回 Master：
- 通道編號 (channel)
- 溫度值 (temperature)
- 時間戳記 (timestamp)
- 耳套狀態 (ear_cover: "1111" 或 "0000")
- 藍芽連線狀態 (bt_state: connected/disconnected/connecting/error)

### 連線建立後的自動同步

1. 網路連線建立時，Slave 自動補送所有通道的藍芽狀態給 Master
2. Master 收到後更新 UI 顯示與 PLC D513 位元
3. Slave 同時傳送通道啟用設定，Master 會比對兩端是否一致
4. 如果 Master 啟用了某通道但 Slave 端停用，會發出「通道不一致」警報

### Slave 資料等待機制

Master 發送量測指令後，會等待 Slave 所有啟用通道的資料都更新（最多等 3 秒）。如果超時仍有通道未回報，系統會在 Log 中記錄「等待 Slave 資料逾時，未收到：{通道名稱}」，並以已收到的資料繼續判定。

## 13. 常見問題

### Q: 啟動後通道一直顯示灰色，藍芽連不上？

請依序檢查：
1. 確認耳溫槍已開機且藍芽已啟用
2. 確認 Windows 藍芽已開啟
3. 確認進階設定中的 MAC 位址正確（格式：`XX:XX:XX:XX:XX:XX`）
4. 確認該通道的啟用開關已打開
5. 查看系統 Log 是否有「連線失敗」的錯誤訊息
6. 嘗試在 Windows 藍芽設定中先手動配對該耳溫槍

### Q: Master 的 Slave 通道全部灰色？

表示 Master-Slave 網路未連線。請確認：
1. Slave 程式是否已啟動
2. 兩端的 Master IP 與 Port 設定是否一致
3. Windows 防火牆是否允許 Port 5001 通過
4. 查看網路指示燈是否為綠色

### Q: PLC 指示燈一直是紅色？

請確認：
1. PLC 是否上電且網路線已連接
2. 確認 PLC IP 位址與 Port 設定正確
3. 可用 `ping {PLC IP}` 測試網路連通性
4. 確認 `config.json` 中 `plc.enabled` 為 `true`

### Q: 如何修改誤差範圍？

1. 點擊右上角齒輪圖示開啟設定面板
2. 輸入管理密碼登入
3. 在「系統設定」中修改「誤差上限」和「誤差下限」
4. 點擊「更新資料（即時套用）」立即生效

### Q: 如何在無硬體環境下測試？

在 `config.json` 中設定 `"simulation_mode": true`，系統會模擬藍芽連線與量測數據。模擬模式下可使用手動觸發按鈕測試完整流程。

### Q: CSV 被 Excel 開啟時系統寫入會失敗嗎？

量測 CSV 使用 append 模式寫入，Excel 開啟時通常可正常寫入。異常記錄 CSV（Alarm）若被鎖定，系統會自動寫入備用檔案（`_1.csv` 後綴），不會遺失資料。

### Q: 停用通道後需要重啟程式嗎？

不需要。修改通道啟用狀態後點擊「更新資料（即時套用）」，系統會立即停止向停用通道發送量測指令、斷開該通道的藍芽連線、清除 PLC 對應的錯誤位元，UI 上以半透明方式顯示。

### Q: 暖槍中空槍值超限會立即報警嗎？

不會。暖槍中（PLC D542=1 時），系統會容忍前 2 次空槍超限，直到連續第 3 次才發出警報。暖槍結束（D542=0）後，超限計數自動歸零。

### Q: 按了 F5 畫面刷新後資料會消失嗎？

按 F5 只會重新載入 UI 畫面，所有後端管理器（藍芽、PLC、網路、量測）仍在運行中。畫面會重新同步到最新狀態。已經寫入 CSV 的資料不受影響。

### Q: 兩台電腦可以同時當 Master 嗎？

不可以。一套系統只能有一個 Master 和一個 Slave。如果兩台都設為 Master，會造成 PLC 寫入衝突與網路連線失敗。
